# 1. Introduction

## System Overview

SylhetStay is a modern web-based hostel booking and management platform designed for students in Sylhet, Bangladesh.

The platform allows users to:

* Browse hostel rooms
* Book accommodations online
* Make secure payments
* Manage profiles and bookings
* View payment history
* Explore facilities and gallery sections

The application is responsive and optimized for desktop, tablet, and mobile devices.

# 6.2 User Registration & Authentication

## Registration Procedure

### Required Information

* Full name
* Email address
* Phone number
* Password

### Process

1. Open Login page
2. Switch to “Sign Up”
3. Complete registration form
4. Submit form
5. Authentication state saved in localStorage
6. Redirect to homepage

---

## Login Procedure

### Required Information

* Email address
* Password

### Process

1. Open Login page
2. Enter credentials
3. Submit form
4. System authenticates user
5. Redirect to homepage

---

## Logout Procedure

1. Open profile dropdown
2. Click “Logout”
3. Authentication state cleared
4. User returned to guest mode

---

# 6.3 Room Booking Procedure

## Step 1: Room Selection

User selects a room and clicks **Book Now**.

---

## Step 2: Booking Information

User provides:

* Check-in date
* Check-out date
* Number of guests
* Personal details
* Special requests (optional)

---

## Step 3: Validation

System validates:

* Required fields
* Email format
* Guest limits
* Date validity

---

## Step 4: Payment Selection

Available methods:

* SSLCommerz
* Credit/Debit Cards
* Mobile Banking
* Net Banking

---

## Step 5: Payment Confirmation

After successful payment:

* Booking is saved
* Transaction ID generated
* Payment added to history
* Booking appears in **My Rooms**

---

# 6.4 Payment Processing Procedure

## Payment Flow

1. Frontend sends payment request
2. Backend communicates with SSLCommerz
3. User redirected to payment gateway
4. Payment processed securely
5. Callback status returned:

   * Success
   * Failed
   * Cancelled

---

## Supported Payment Methods

* Visa
* MasterCard
* American Express
* bKash
* Nagad
* Rocket
* Upay
* Net Banking

# 7. System Workflows

# 7.1 Complete Booking Workflow

1. User browses rooms
2. User selects room
3. Authentication check performed
4. Booking form completed
5. Payment method selected
6. Payment processed
7. Booking confirmed
8. Booking & payment saved

---

# 7.2 Payment Gateway Workflow

1. Frontend initiates payment
2. Backend sends request to SSLCommerz
3. Gateway processes payment
4. Callback response returned
5. Frontend displays payment status

---

# 7.3 Authentication Workflow

1. User registers or logs in
2. System validates input
3. Authentication stored in localStorage
4. User gains access to protected pages

# 9. Security Procedures

## Current Security Measures

* Client-side form validation
* Secure SSLCommerz integration
* Environment variable protection
* CORS middleware support

---

## Current Limitations

* Authentication is prototype-based
* No backend password hashing
* No JWT/session authentication
* localStorage vulnerabilities

---

## Recommended Enhancements

* Implement JWT authentication
* Add bcrypt password hashing
* Enforce HTTPS
* Add rate limiting
* Implement CSRF protection

# 10. Maintenance & Troubleshooting

## Routine Maintenance

* Update dependencies regularly
* Monitor payment gateway credentials
* Check image assets
* Review build performance

---

## Common Issues

### Payment Server Not Starting

Possible causes:

* Port conflict
* Missing dependencies
* Incorrect environment variables

---

### Booking Not Saving

Possible causes:

* localStorage limit exceeded
* JavaScript errors
* Callback issues

---

### Authentication Issues

Possible causes:

* Cleared browser data
* Incognito mode restrictions
* localStorage disabled

---

### Deployment Issues

Possible causes:

* Build errors
* Missing environment variables
* Incorrect Vercel configuration

# 11. Deployment Procedures

# Development Setup

## Step 1: Clone Repository

```bash
git clone <repository-url>
cd project-folder
```

## Step 2: Install Dependencies

```bash
npm install
```

## Step 3: Configure Environment Variables

```env
SSLCOMMERZ_STORE_ID=your_store_id
SSLCOMMERZ_STORE_PASSWORD=your_store_password
NODE_ENV=development
PORT=3001
```

---

## Step 4: Run Development Server

```bash
npm run dev
```

---

# Production Deployment

## Build Application

```bash
npm run build
```

---

## Deploy to Vercel

* Connect repository to Vercel
* Add environment variables
* Deploy production build

---

## Recommended Post-Deployment Checks

* Verify room browsing
* Test booking flow
* Test authentication
* Verify payment callbacks
* Check responsive layout
* Test dark/light mode
